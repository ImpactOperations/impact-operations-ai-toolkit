# prompt-builder

**A Claude skill that turns vague task descriptions into ready-to-use AI prompts.**

---

## What this does

Getting useful results from AI tools often fails not because the task is hard, but because the request was fuzzy. This skill takes what you want to achieve and turns it into a clear, structured prompt you can copy and paste directly into Claude.

---

## When to use it

- Use this when you keep getting generic or unhelpful responses from Claude and aren't sure what to change
- Use this when you need to brief Claude on a recurring task (like writing grant reports or board updates) and want something you can reuse
- Use this when you know what you want to produce but aren't sure how to ask for it
- Use this when you've got a high-stakes document (a funding application, an RFP response, a board paper) and you want Claude to do it well, not just adequately
- Use this when a colleague wants to start using Claude but doesn't know where to begin

---

## How to use it

1. Open a new Claude conversation with this skill active
2. Describe the task you want to accomplish in plain language, for example: "I need Claude to help me write a funding acquittal report" or "help me ask Claude to summarise staff survey results"
3. Paste any relevant context if you have it (a document, a brief, a job description)
4. Receive your ready-to-use prompt and follow the instructions to refine it if needed

---

## What you'll get back

You will receive a finished prompt in a copy-ready block, a short note on why it is structured that way, and a suggestion for which version of Claude to run it on. Take the prompt, paste it into a new Claude conversation, add any source material it asks for, and run it.

---

## Good to know

- This skill produces the prompt, not the final document. That is intentional: once you have a good prompt, you can reuse it across projects and adapt it yourself.
- For anything involving financial data, legal matters, or sensitive participant information, the prompts include a reminder to have a human review the output before using it.
- The more context you give at the start (your audience, what good looks like, what you are trying to avoid), the sharper the prompt will be.
